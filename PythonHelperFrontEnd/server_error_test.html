🔧 服务器错误处理测试

本页面用于测试改进后的错误处理机制

📊 当前错误类型说明

服务器日志显示：
INFO:app.email_service:邮件已发送到: rzzhang@zju.edu.cn
ERROR:app.routes.auth_routes:发送验证码失败: no such table: pending_verifications
INFO:werkzeug:172.25.144.1 - - [04/Oct/2025 22:20:58] "POST /auth/send-verification HTTP/1.1" 500 -

错误分析：

  • 🔍 错误类型：HTTP 500 Internal Server Error
  • 🗄️ 根本原因：数据库缺少 pending_verifications 表
  • 📧 邮件状态：邮件发送成功，但验证码存储失败
  • 🛠️ 需要修复：后端数据库表结构

✅ 前端错误处理改进

针对不同类型的错误，前端现在会显示更友好的错误信息：

HTTP 500 错误： 服务器内部错误，数据库配置问题，请联系管理员
HTTP 404 错误： 服务不可用，请检查服务器状态
网络错误： 网络连接失败，请检查网络后重试
其他HTTP错误： 服务器错误：[具体错误信息]

🔧 后端修复建议

要彻底解决这个问题，需要在后端进行以下修复：

  1. 创建缺失的数据库表：
    CREATE TABLE pending_verifications (
      id INTEGER PRIMARY KEY AUTOINCREMENT,
      email TEXT NOT NULL,
      verification_code TEXT NOT NULL,
      type TEXT NOT NULL,
      created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
      expires_at TIMESTAMP NOT NULL
    );
  2. 检查数据库初始化脚本
  3. 确保所有必要的表都已创建
  4. 添加数据库迁移机制
